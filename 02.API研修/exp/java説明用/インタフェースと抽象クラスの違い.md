## 抽象クラス

### 抽象クラスの特徴

- abstract class で定義
- インスタンス化できない（new 不可）
- フィールド（状態）やコンストラクタを持てる
- 共通の処理を実装できる
- 継承は 1 つだけ（単一継承）

- 使う場面

  - 継承関係（IS-A）を表現したいとき
    例：Dog IS-A Animal
    ```java
    abstract class Animal {
        String name;
        public void sleep() {
        System.out.println("寝る");
    }
        public abstract void speak();
    }
    ```
    ⇒ クラス実装側に実装方法を促す役割？

## インタフェース

### インタフェースの特徴

- interface で定義
- 実装クラスにメソッドの定義だけを強制（契約）
- 状態（フィールド）は持てない（定数のみ）
- 多重実装が可能（複数のインタフェースを同時に実装できる）

- 使う場面

  - クラスに特定の機能（CAN-DO）を持たせたいとき
  - 実装の詳細には触れず、外部との接点や仕様を定義したいとき
    例：Printer CAN Printable
    ```java
    interface Printable {
        void print();
    }
    ```
    ⇒ クラスを使う側にどんな機能があるかを知らせる役割

### 例え

- 抽象クラス：動物という“共通の性質”を持つ設計図（寝る、名前を持つなど）
- インタフェース：レジが“会計できる”という機能の契約（クレジットでも現金でも OK）
