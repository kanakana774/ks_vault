# API とは

**API（Application Programming Interface）** とは、ソフトウェアの機能やデータを、別のソフトウェアから利用するための仕組みや窓口のことです。

プログラミングにおいて、あるソフトウェアの機能を利用したい場合、そのソフトウェアの内部構造をすべて理解する必要はありません。API が提供する決められた手順や規約に従ってリクエストを送るだけで、必要な機能やデータにアクセスできるようになります。

例えるなら、レストランで食事をする際に、料理の作り方をすべて知らなくても、メニュー（API）を見て注文（リクエスト）すれば、料理（機能やデータ）が運ばれてくるようなものです。

API には、以下のようなメリットがあります。

- **開発効率の向上:** ゼロからすべてを開発する必要がなく、既存の機能やサービスを組み合わせて新しいアプリケーションを素早く開発できます。
- **機能の拡張性:** 外部のサービスが提供する豊富な機能を、自分のアプリケーションに容易に取り込むことができます。
- **システムの連携:** 異なるシステム間でのデータ共有や機能連携が容易になります。

身近な例としては、以下のようなものがあります。

- **Google Maps API:** 自分のウェブサイトに Google マップを埋め込んだり、ルート検索機能などを利用したりできます。
- **SNS の API:** Twitter や Facebook などの投稿機能やユーザー情報を、他のアプリケーションから利用できます。
- **天気予報 API:** 天気予報のデータを取得して、自分のアプリケーションに表示できます。

これらの API を通じて、様々なアプリケーションやサービスが連携し、より便利で豊かなデジタル体験が生まれています。

## 補足

実は java の List インタフェースなども広義には API です。なので、前述の API は詳細には webAPI と表現するのが的確です。\
例えば add()メソッドの実装の中身を知らなくても、下記のように使えば、list に要素を追加できることを知っていれば使えますよね。\
インタフェース（＝物の境界）は窓口の役割を担っていて、そこに頼めば内部の処理は分からなくても使えるというメリットがあります。

```java
List<String> list = new ArrayList<>();
list.add("hoge");
```

# 一般公開されてる API を呼んでみる

天気を知ることができる API を呼んでみる。

openweathermapAPI の仕様書：
https://openweathermap.org/api

```
http://api.openweathermap.org/data/2.5/weather?q={city}&appid={key}&lang=ja&units=metric

curl "http://api.openweathermap.org/data/2.5/forecast?q=Tokyo,JP&appid=ae81af837c57a19a9d85e5e682158d92&lang=ja&units=metric"
```
