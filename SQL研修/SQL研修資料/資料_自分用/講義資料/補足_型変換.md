### 型定数

[https://www.postgresql.jp/docs/9.4/sql-syntax-lexical.html#SQL-SYNTAX-CONSTANTS-GENERIC:~:text=%E3%81%AA%E5%A0%B4%E5%90%88%E3%81%A7%E3%81%99%E3%80%82-,4.1.2.7.%20%E4%BB%96%E3%81%AE%E5%9E%8B%E3%81%AE%E5%AE%9A%E6%95%B0,-%E4%BB%BB%E6%84%8F%E3%81%AE%E5%9E%8B]

定数は下記いづれかの記載方法があり、ほかの RDBMS と共通の cast を使うのが丸いかな
type は型のことで、date など

```SQL
type 'string'
'string'::type
CAST ( 'string' AS type )
```

例えば、interval '3 day'などは type 'string'の構文を使っていて、 '3 day'を interval 型に変換してる。

[https://www.postgresql.jp/document/17/html/datatype-datetime.html#DATATYPE-DATETIME-INPUT:~:text=%E3%82%89%E3%82%8C%E3%81%BE%E3%81%99%E3%80%82-,8.5.4.%20%E6%99%82%E9%96%93%E9%96%93%E9%9A%94%E3%81%AE%E5%85%A5%E5%8A%9B,-%23]

```SQL
interval '[@] quantity unit [quantity unit...] [direction]'
```

quantity が 1 とかで、unit が month とか
