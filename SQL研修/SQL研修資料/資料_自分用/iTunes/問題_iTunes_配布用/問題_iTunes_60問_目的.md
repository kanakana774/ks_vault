
### 1. アピールの核となる重要演習問題

以下の問題は、基本的なスキルから応用的な分析スキルまでをバランス良くカバーしており、面接で「何をしてきたか」を具体的に語るのに最適です。

| カテゴリ              | 特徴・アピールポイント                                   | 関連する重要問題                                               |
| :---------------- | :-------------------------------------------- | :----------------------------------------------------- |
| **基礎的なデータ集計**     | 複数テーブルを結合し、ビジネスに必要な基本指標を算出できることを示す。           | **問題1:** アーティストごとのアルバム数と曲数                             |
| **ビジネス分析（ランキング）** | CTEとウィンドウ関数を使い、複雑な要件を整理してランキングを作成できる。         | **問題6:** 各国で最も売れたジャンル<br>**問題20:** ジャンル別で最も売れたトラック     |
| **顧客行動分析**        | ウィンドウ関数を使い、顧客一人ひとりの時間軸に沿った分析ができる。             | **問題12:** 顧客ごとの購入間隔と累積購入額<br>**問題32:** 国別ランキングと売上比率    |
| **条件に基づくデータ抽出**   | `NOT EXISTS` 等を使い、「〜していない顧客」といった実務で頻出の抽出ができる。 | **問題14:** 特定期間に購入がない顧客（休眠顧客）                           |
| **グループ内での比較・分析**  | `PARTITION BY` を活用し、グループ内での比率や平均との比較ができる。     | **問題16:** アーティスト内でのアルバム売上比率<br>**問題21:** 国の平均購入額より高い顧客 |
| **時系列データ分析**      | `LAG`関数を使い、前期比較など、時間的な変化を捉える分析ができる。           | **問題37:** 四半期ごとの売上と前年同期比                               |

これらの問題に重点を置くことで、研修生は自信を持って面接で語れる「引き出し」を持つことができます。

### 2. 口頭でのアピール方法

IT未経験という前提を踏まえ、謙虚さを示しつつも、学んだことを具体的に伝えることを目指します。

**アピールトークの例：**

「1ヶ月間のSQL研修では、まず`SELECT`文や`JOIN`によるテーブル結合といった基礎から学びました。研修の後半では、10以上のテーブルが正規化されたデータベースを使い、より実践的なデータ抽出の課題に取り組みました。

例えば、**『各国の顧客の購入額ランキングと、その国の総売上に対する各顧客の売上比率を算出する』**（問題32）といった課題です。この課題では、まずCTE（共通テーブル式）を使って顧客ごとの購入額を先に集計し、複雑なクエリを分かりやすく整理しました。その上で、`RANK()`というウィンドウ関数で国ごとに順位を付け、`SUM() OVER()`で国ごとの総売上を算出して比率を計算しました。

また、**『前回購入時からの経過日数を計算する』**（問題12）という課題では、`LAG()`というウィンドウ関数を使い、顧客ごとに時系列でデータを扱う方法を学びました。

これらの演習を通して、ビジネス上の要求をどのようにSQLで実現するかの手順を考える力が身につきました。まだ実務経験はありませんが、これらの経験を活かして、まずは既存のSQLを正確に読み解き、簡単な修正から担当させていただくことで、一日も早く現場に貢献したいと考えております。」

**このトークのポイント：**

*   **地に足のついた表現：** 「価値を提供できる」から「一日も早く現場に貢献したい」という意欲の表現に留めることで、未経験者としての謙虚な姿勢を示します。
*   **技術用語と目的をセットで説明：** 「CTEでクエリを整理」「`RANK()`で順位付け」のように、技術が「何のために」使われるのかを簡潔に説明することで、丸暗記ではないことをアピールします。
*   **具体的な問題番号との紐付け：** 講師として、研修生がこのトークをする際に、どの問題に基づいているかを明確に指導できます。

### 3. スキルシートへのアピール方法（リファレンス付き）

各スキル項目が、どの演習問題を解くことで証明されるのかを明記します。これにより、スキルシートの記述に説得力を持たせることができます。

**件名：SQL（PostgreSQL）を用いたデータ抽出・分析スキル（1ヶ月間の実践研修）**

*   `SELECT`, `JOIN`, `GROUP BY` 等の基本構文を習熟し、要件定義に基づいたデータ抽出・集計が可能。
    *   （関連問題: 問題1, 問題4）
*   正規化された複数テーブル構造を理解し、外部キーを用いてデータを結合した横断的な集計・抽出経験。
    *   （関連問題: 問題1, 問題2, 問題5）
*   CTE（共通テーブル式）を利用し、ステップごとに処理を分割することで可読性の高いSQLを記述可能。
    *   （関連問題: 問題6, 問題7, 問題16, 問題21）
*   ウィンドウ関数を用いた高度な集計・分析SQLの作成経験。
    *   **ランキング作成** (`ROW_NUMBER`, `RANK`): 問題6, 問題8, 問題20, 問題32
    *   **累積合計・グループ内比率** (`SUM OVER`): 問題3, 問題16, 問題32
    *   **時系列比較** (`LAG`): 問題12, 問題37
*   ビジネス要件をSQLクエリに変換する実践的な課題解決経験。
    *   **休眠顧客の抽出** (`NOT EXISTS`): 問題14
    *   **特定期間での購入者絞り込み** (`INTERSECT`): 問題17
*   `EXTRACT`や`DATE_TRUNC`等の関数を用いた日付・時刻データの加工経験。
    *   （関連問題: 問題2, 問題7, 問題24, 問題37）
*   サブクエリ（特に相関サブクエリ）を用いて、行ごとに動的な条件でのデータ抽出が可能。
    *   （関連問題: 問題8, 問題10）
*   `UNION`, `INTERSECT`, `EXCEPT`といった集合演算子を適切に使い分ける知識。
    *   （関連問題: 問題17, 問題22, 問題33）
*   `HAVING`句を用いた、集計結果に対する条件指定が可能。
    *   （関連問題: 問題5, 問題9, 問題14）
